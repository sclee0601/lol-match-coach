# Challenger Behavior - Situational Decision Guide

## Wave Management Decisions
- **Wave crashing into enemy tower** -> Roam, invade, or set up objective. Never stand idle.
- **Wave pushing toward you** -> Do not leave lane. Clear it first or you lose CS and tower.
- **Before a major objective (60-90s out)** -> Slow push your wave so it crashes while you fight the objective.
- **After winning a fight** -> Immediately convert: push tower, take objective, or invade. Standing still after a kill is a tempo loss.
- **After losing a fight** -> Reset wave state before recalling if possible.

## Objective Decisions
- **Dragon spawning** -> Be in position 30 seconds before spawn. Vision setup starts 90 seconds before.
- **Baron spawning** -> Vision setup at 19:00. Full team grouping by 19:30.
- **After winning a teamfight near an objective** -> Take the objective immediately. Do not recall, do not chase.
- **Rift Herald** -> Use it to destroy towers mid for maximum pressure.

## Positioning Decisions
- **Low HP in a contested zone** -> Trace back to the decision that put you there.
- **ADC in teamfight** -> Maximum attack range at all times.
- **Support roaming** -> Only roam when wave is crashed into enemy tower.
- **Jungler pathing** -> Always have a 3-camp plan before recalling.

## Trading & Fighting Decisions
- **All-in window** -> Enemy used gap closer, shield, or heal? Commit.
- **Short vs extended trade** -> Know your champion's trade pattern.
- **Diving** -> Only dive with numbers advantage AND good wave state.

## Death Analysis Framework
Every death traces back to one of these root causes:
1. **Wave management error** -> Bad wave state forced a bad position
2. **Vision failure** -> No information led to getting caught
3. **Cooldown tracking failure** -> Engaged when enemy had key abilities
4. **Macro error** -> Wrong place at wrong time
5. **Mechanical error** -> Correct decision, wrong execution
